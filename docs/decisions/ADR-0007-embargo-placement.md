# ADR-0007 — Embargo placement: stacked form

## Context

Purged k-fold and CPCV split the observation sequence into K folds.  For each
test fold `[b_start, b_end)`, three regions are excluded from training:

1. **Pre-purge**: `[b_start − purge, b_start)` — labels from the period
   immediately before the test block may overlap into it.
2. **Post-purge**: `[b_end, b_end + purge)` — symmetric label-horizon gap.
3. **Embargo**: a further buffer beyond the post-purge region that guards
   against serial-correlation bleed (independent of label horizon).

Two placement conventions exist for the embargo:

| Convention | Embargo start | Embargo end | Total trailing excluded band |
|---|---|---|---|
| **Overlap** | `b_end` | `b_end + embargo` | purge and embargo overlap; effective = `max(purge, embargo)` |
| **Stacked** | `b_end + purge` | `b_end + purge + embargo` | total = `purge + embargo` |

The distinction matters only when `embargo < purge`; in walk-forward
(training is entirely left of the test block) the embargo simply occupies
`[test_end, test_end + embargo)` and there is no stacking question.

## Decision

**Use the stacked form.**

Embargo begins at `b_end + purge` and ends at `b_end + purge + embargo`.

## Rationale

### 1. Primary source — López de Prado 2018 AFML Snippet 7.3

López de Prado's reference implementation (`ml_get_train_times`,
reproduced in AFML pp. 105-106, Wiley 2018 print) constructs the excluded
interval by first extending the test interval's right edge by the embargo,
**then** applying the label-horizon purge.  The resulting trailing excluded
band is `[b_end, b_end + purge + embargo)` — the stacked form.

### 2. mlfinlab 1.6.0 reference implementation

The open-source `mlfinlab` library (Hudson & Thames, commit history
consistent with AFML Snippet 7.3) implements `ml_get_train_times` with the
stacked form.  Downstream tooling in the sibling SKIE_Ninja research repo
(CPCV + PBO walk-forward) consumes splits generated by `ml_get_train_times`
conventions; deviating would produce incompatible fold boundaries when the
sibling-repo prior screen is compared against this project's walk-forward
results.

### 3. Conservative coverage

The stacked form always excludes at least as many observations as the overlap
form (the excluded band is strictly wider when both purge > 0 and embargo > 0).
It is therefore the conservative choice: it reduces training-set size slightly
but never understates the required gap.

### 4. Cycle 4 implementation precedent

The Round-1 implementation of `purged_kfold_split` used the overlap form.
The Round-2 F-2-1 finding in the Cycle 4 audit explicitly reverted this to
the stacked form to match the mlfinlab convention.  The current code comment
in [src/skie_ninja/backtest/splits.py](../../src/skie_ninja/backtest/splits.py)
at line ~552 records this history inline.

## Ambiguity documented

The overlap form is methodologically defensible: if the post-purge band already
clears the label horizon, adding the embargo on top of it (stacked) versus
overlapping it (overlap) does not affect label leakage — only serial-correlation
bleed.  The stacked form is strictly wider, so it trades a small amount of
training data for consistency with the canonical reference implementation.

For H050's `purge = max(vertical_barrier)` (≤ 120 minutes) and
`embargo = max(PACF lag, PW-block length)` (typically ≤ 30 minutes), the
stacked band is at most 150 minutes per fold boundary — negligible relative
to multi-year training windows.

## Consequences

- `purged_kfold_split` and `cpcv_split` in `splits.py` implement stacked;
  no code change required.
- `walk_forward_split` is unaffected (training is always left of test).
- Any sibling-repo CPCV + PBO comparison must document that it also uses
  stacked; document in the hypothesis register cross-reference when H050 SPA
  universe grows.
- `P1-BACKTEST-EMBARGO-MODE-ADR` is closed by this ADR.
